# 2. 想定ユーザー体験

[設計書トップへ戻る](../DESIGN.md)

## 概要

ユーザーは普段通りChromeでページを開き、Claudeに依頼するだけでページ内容の取得や操作を行える。
理想は、Chrome拡張を入れて最初の簡単な案内に従うだけでMCPが利用可能になる体験にする。
ローカルサーバー起動、別ブラウザ起動、ログインし直し、Chromeプロファイル変更を意識しない体験にする。

## 最終ゴール

初回セットアップは、拡張パネルの案内を中心にして、Goヘルパー配置、MCP登録、ペアリングを可能な限り自動化する。
認証トークンの手入力、認証情報リセット、設定ファイル編集、プロファイル切り替えを通常操作に含めない。
失敗時は「何が起きたか」と「何をすれば復旧できるか」を返し、拡張側の案内で自然に復旧できるようにする。

## 詳細化する項目

- 初回セットアップの流れ
- 拡張パネルでの接続状態表示
- ペアリング、再接続、トークン再発行をユーザーに意識させないUX
- Goヘルパー配置とMCP登録の自動化
- 失敗時にClaudeへ返す案内文
